Amdahl’s Law

隐藏的serial点：

  1. queue.take()
  2. 代码之外的result handling，要么有地方汇总结果，要么有side effect
    1. side effect，比如写日志、写数据库
 1public class WorkerThread extends Thread {
 2  private final BlockingQueue<Runnable> queue;
 3  public void run() {
 4    while (true) {
 5      try {
 6        Runnable task = queue.take(); // <- serial点
 7        task.run();
 8      } catch (InterruptedException e) {
 9        break;
10      }
11    }  
12  }
13}

线程的代价

优化非竞争性锁

某个锁被证明不会发生竞争，JVM会优化掉它：

1synchronized (new Object()) {
2  // do something
3}

通过逃逸分析发现对象从未跳出当前线程，JVM会优化掉它：

1public String getStoogeNames() {
2  List<String> stooges = new Vector<>();
3  stooges.add("foo");
4  stooges.add("bar");
5  return stooges.toString();
6}

上述代码就算不用逃逸分析，编译器也会合并相邻的锁。

优化竞争性锁

缩小锁的范围

优化前：

1private HashMap<String, String> attributes;
2public synchronized boolean userLocationMatches(String name, String regexp) {
3  String key = "users." + name + ".location";
4  String location = attributes.get(key);
5  return Pattern.matches(regexp, location);
6}

优化后：

1private HashMap<String, String> attributes;
2public boolean userLocationMatches(String name, String regexp) {
3  String key = "users." + name + ".location";
4  String location;
5  synchronized (this) {
6    location = attributes.get(key);
7  }
8  return Pattern.matches(regexp, location);
9}

或者：

1private Map<String, String> attributes = Collections.synchronizedMap(...);

Lock Splitting，锁分割

不要用一把锁保护两个独立的状态变量：

 1public class ServerStatus {
 2  private Set<String> users;
 3  private Set<String> queries;
 4  public synchronized void addUser(...) {
 5    users.add(...);
 6  }
 7  pubilc synchronized void addQuery(...) {
 8    queries.add(...);
 9  }
10}

可以两把锁来会更好：

 1public class ServerStatus {
 2  private Set<String> users;
 3  private Set<String> queries;
 4  public void addUser(...) {
 5    synchronized (users) {
 6      users.add(...);      
 7    }
 8  }
 9  pubilc void addQuery(...) {
10    synchronized (queries) {
11      queries.add(...);      
12    }
13  }
14}
